API Overview

The Equipment Management System provides a comprehensive RESTful API for programmatic access to equipment data, maintenance records, inspections, and more.

Key Features:
  • Full CRUD Operations - Create, Read, Update, Delete
  • Filtering & Search - Find exactly what you need
  • Pagination - Efficient data retrieval
  • Nested Data - Related objects included
  • Custom Actions - Specialized endpoints
  • Authentication Required - Secure access only

Available Endpoints

Endpoint Description Methods
/api/equipment/ Equipment management GET POST PUT DELETE
/api/manufacturers/ Manufacturer data GET POST PUT DELETE
/api/categories/ Equipment categories GET POST PUT DELETE
/api/maintenance/ Maintenance records GET POST PUT DELETE
/api/maintenance-types/ Maintenance types GET Read Only
/api/inspections/ Inspection records GET POST PUT DELETE
/api/profiles/ User profiles GET Read Only

Authentication

The API supports two authentication methods:

1. Session Authentication (Browser)

When logged in through the web interface, API requests are automatically authenticated.

2. Basic Authentication (External Apps)
curl -u username:password http://127.0.0.1:8000/api/equipment/
Note: All API endpoints require authentication. Anonymous users will receive a 403 Forbidden response.

Usage Examples

Python Example:
import requests

# Get all equipment
response = requests.get(
    'http://127.0.0.1:8000/api/equipment/',
    auth=('username', 'password')
)
equipment = response.json()
print(f"Total equipment: {equipment['count']}")

# Search for specific equipment
response = requests.get(
    'http://127.0.0.1:8000/api/equipment/',
    params={'search': 'balance'},
    auth=('username', 'password')
)
results = response.json()['results']
JavaScript Example:
// Fetch equipment data
fetch('http://127.0.0.1:8000/api/equipment/', {
    headers: {
        'Authorization': 'Basic ' + btoa('username:password')
    }
})
.then(response => response.json())
.then(data => console.log(data));
